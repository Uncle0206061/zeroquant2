# 任务卡格式规范

每张任务卡必须包含以下字段，缺一不可。

## 必填字段

| 字段 | 说明 |
|------|------|
| 任务编号 | TC-XXX-{终端}，如 TC-M1-01-niuniu |
| 任务名称 | 简短描述 |
| 任务类型 | 代码编写 / 配置搭建 / 接口开发 / 联调 / 文档 / 测试 |
| 截止时间 | YYYY-MM-DD HH:MM |
| 前置依赖 | 依赖的任务卡编号，完成后才能开始 |
| 所属终端 | 哪个终端负责 |
| 所属里程碑 | M1~M6 |
| 优先级 | 🔴 高 / 🟡 中 / 🟢 低 |

## 任务内容

具体步骤，每个步骤清晰可执行，无歧义。

## 交付物清单

每个交付物用 `[ ]` 标记，完成后改为 `[x]`。

## 审核点

每条审核点必须可验证（能实际检查通过/不通过）。

## 反馈要求

说明遇到问题怎么办、完成时怎么提交。
