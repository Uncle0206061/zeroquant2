## 二、执行计划

### Step 1: 三端架构理解汇报

**三端职责边界：**

| 终端 | 技术栈 | 端口 | 职责 |
|------|--------|------|------|
| 妞妞（Go后端） | Go + Gin + PostgreSQL + Redis + WebSocket | 8080 | 用户认证、策略管理、交易执行、持仓管理、WebSocket 推送 |
| 大聪明（Vue3前端） | Vue3 + Element Plus + Vant + ECharts | 5173 | PC/移动端界面、用户交互、实时数据展示 |
| 数据官（Python） | FastAPI + akshare + Tushare Pro | 8081 | 行情采集、Level2 盘口、K线数据、板块数据 |

**前端调用后端 API 规范：**
- Base URL: `http://localhost:8080/api/v1/`（Go 后端）
- 数据服务 URL: `http://localhost:8081/data/v1/`（Python 数据服务）
- 请求头：所有请求携带 `Authorization: Bearer {JWT_TOKEN}`
- 响应格式：`{ "code": 0, "message": "success", "data": {...} }`

**前端获取实时行情数据：**
- WebSocket 地址：`ws://localhost:8080/ws`
- 心跳间隔：10 秒
- 断开重连：最多 10 次，间隔 2 秒

### Step 3: 创建前端项目目录

在本地创建前端项目：
- 本地路径：`D:\ZeroQuant2\frontend\`
- NAS 同步路径：`Z:\ZeroQuant2\src\frontend\`

### Step 4: 初始化 Vue3 项目

使用 Vite 创建 Vue3 + TypeScript 项目：
```bash
npm create vite@latest frontend -- --template vue-ts
cd frontend
pnpm install
```

### Step 5: 安装核心依赖

```bash
pnpm add axios pinia vue-router element-plus vant echarts
pnpm add -D @types/node sass
```

### Step 6: 创建目录结构

按技术规范创建目录：
```
src/
├── api/           ← API 请求封装
├── assets/        ← 静态资源
├── components/    ← 公共组件
├── composables/   ← 组合式函数
├── layouts/       ← 布局组件
├── router/        ← 路由配置
├── stores/        ← Pinia 状态
├── utils/         ← 工具函数
└── views/         ← 页面组件
```

### Step 7: 配置基础文件

- `vite.config.ts` — 配置代理、别名
- `src/api/backend.ts` — Axios 封装 /api/v1/
- `src/api/data.ts` — Axios 封装 /data/v1/
- `src/utils/websocket.ts` — WebSocket 封装
