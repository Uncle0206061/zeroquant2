## 任务内容

### 1. WebSocket 连接
```
WS /api/v1/ws
Header: Authorization: Bearer {token}
```
- JWT 鉴权后才可连接
- 全局唯一 WebSocket 实例（Goroutine 管理）
- 连接时注册到用户 session map

### 2. 心跳机制（技术规范强制）
- 心跳间隔：每 10 秒发送 ping
- 客户端 10 秒内无响应则断开
- 超过最大重连次数（10 次）后停止重连，间隔 2 秒

### 3. 推送事件类型
| event | 触发时机 | 数据内容 |
|-------|---------|---------|
| position_update | 持仓变化 | { positions: [...] } |
| order_update | 订单状态变化 | { orders: [...] } |
| backtest_progress | 回测进行中 | { task_id, progress, status } |
| backtest_result | 回测完成 | { task_id, result: {...} } |
| strategy_signal | 策略信号 | { strategy_id, signal, price } |

### 4. 推送延迟控制
- 推送延迟 ≤ 200ms
- 批量推送时合并消息，减少网络开销

### 5. 重连机制（客户端视角，Go 服务端配合）
- 断线自动重连：最多 10 次，间隔 2 秒
- 重连后补推送未读消息（保留最近 100 条）
