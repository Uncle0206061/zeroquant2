## 任务内容

### 1. 策略 CRUD
```
GET    /api/v1/strategy/list      ← 列表（分页）
POST   /api/v1/strategy/create    ← 创建
GET    /api/v1/strategy/:id       ← 详情
PUT    /api/v1/strategy/:id       ← 更新
DELETE /api/v1/strategy/:id       ← 删除
```
- 策略表：biz_strategy、biz_strategy_rule
- 创建时写规则表，关联策略ID

### 2. 策略规则结构
策略规则 JSON 字段结构（参考需求文档 REQ-S004）：
- 选股条件（财务指标、技术指标、板块）
- 择时条件（买入时机、止损止盈）
- 风控参数（最大仓位、单笔上限）
- 支持前端策略编辑器输出的格式

### 3. 提交执行
```
POST /api/v1/strategy/:id/submit
Body: { backtest_mode: "realtime" | "historian" }
响应：{ code: 0, data: { task_id, status: "pending" } }
```
- 提交成功后写 biz_backtest 表
- 异步处理，WebSocket 推送结果

### 4. 调用数据服务
```
POST /data/v1/filter/
Body: { rules: [...], stocks: [...] }
```
- 策略筛选时调用 Python 数据服务的 filter 接口
- 超时时间：5 秒，超时返回错误
