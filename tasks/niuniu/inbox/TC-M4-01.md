## 任务内容

### 1. 模拟账户
```
POST /api/v1/account/simulate/create
响应：{ code: 0, data: { account_id, balance: 1000000 } }
```
- 每个用户一个模拟账户（初始资金 100 万）
- 资金余额表：biz_portfolio

### 2. 订单管理
```
POST /api/v1/order/submit    ← 下单
GET  /api/v1/order/list      ← 订单列表（分页，支持状态过滤）
GET  /api/v1/order/:id       ← 订单详情
DELETE /api/v1/order/:id     ← 撤单（状态 pending 才可撤）
```
订单表：biz_order（status: pending/filled/cancelled/rejected）

### 3. 持仓管理
```
GET /api/v1/position          ← 当前持仓列表
GET /api/v1/position/:stock_code ← 单股持仓
```
- 实时计算持仓成本、盈亏金额、盈亏比例
- 持仓表：biz_position

### 4. 撮合引擎
- 实时撮合（模拟成交）
- 买入：价格 ≤ 盘口卖一价 → 成交
- 卖出：价格 ≥ 盘口买一价 → 成交
- 成交后写 biz_order（status=filled）+ biz_position 更新
- WebSocket 推送 order_update + position_update

### 5. 风控规则（技术规范强制）
| 规则 | 参数 |
|------|------|
| 最大仓位 | 单股 ≤ 总资金 30% |
| 最大亏损 | 日亏损 ≥ 5% 自动冻结当日交易 |
| 交易次数 | 日内交易次数 ≤ 50 |
| ST 限制 | *ST 和 ST 股禁止买入 |

### 6. 禁止并发（技术规范强制）
- 下单接口：串行执行（Mutex 锁），禁止并发
- 并发测试：同时 10 请求下单 → 最终成交数量 ≤ 1
