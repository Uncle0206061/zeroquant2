## 任务内容

### 1. 监控告警系统
实现 /data/v1/monitor/status 接口，提供数据服务质量监控：

```
GET /data/v1/monitor/status
响应：{
  "data_sources": { "akshare": "ok"|"error", "tushare": "ok"|"error" },
  "redis_health": true|false,
  "postgres_health": true|false,
  "collector_running": { "orderbook": true|false, "tick": true|false, "kline": true|false },
  "last_heartbeat": "2026-05-06T09:00:00Z"
}
```

告警触发条件：
| 告警级别 | 条件 | 通知方式 |
|---------|------|---------|
| 🔴 紧急 | 全部数据源失效 > 5 分钟 | WebSocket alert + 日志 |
| 🔴 紧急 | PostgreSQL 连接断开 > 1 分钟 | WebSocket alert + 日志 |
| 🟡 警告 | 单数据源失效 > 30 秒 | 日志记录 |
| 🟡 警告 | 某股票 K 线数据缺失 > 5 分钟 | 日志记录 |
| 🟢 提示 | 数据切换源 | 日志记录 |

### 2. 数据质量指标
计算并暴露以下指标（供 Go 后端 /data/v1/monitor/metrics 调用）：

| 指标 | 说明 | 计算方式 |
|------|------|---------|
| 数据完整性 | K 线缺口比例 | 缺口分钟数 / 应有分钟数 |
| 实时延迟 | 盘口数据延迟 | 当前时间 - 数据时间戳 |
| 数据源可用率 | 各源可用时间比例 | 可用时长 / 总时长 |
| 采集吞吐 | 每分钟采集记录数 | 计数器 |

```
GET /data/v1/monitor/metrics
响应：{
  "data": {
    "kline_completeness": 0.998,
    "orderbook_latency_ms": 120,
    "datasource_uptime": { "akshare": 0.999, "tushare": 0.997 },
    "collector_throughput_per_min": 1500
  }
}
```

### 3. 备份机制
- **每日凌晨 3:00 自动备份**：data_kline、data_quote 导出为 CSV 文件
- **备份路径**：`Z:\ZeroQuant2\backup\data\{date}\`（NAS 持久化）
- **保留策略**：最近 30 天
- **备份清单文件**：`backup_manifest.json`（含备份时间、文件列表、MD5 校验值）
- **Redis 持久化**：配置 AOF 持久化（appendfsync everysec）

### 4. 恢复机制
- 断电 / 崩溃后，数据服务启动时自动检测并补全缺失数据
- K 线补全：从 akshare 拉取缺失区间数据
- 盘口数据：仅实时，无历史恢复（Redis 重放）
- 恢复完成后写日志并 WebSocket 通知
