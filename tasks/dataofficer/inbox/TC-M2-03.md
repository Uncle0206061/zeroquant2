## 任务内容

### 1. 多因子筛选接口
实现 POST /data/v1/filter/，接收 Go 后端转发的策略筛选请求：

```
POST /data/v1/filter/
Body: {
  "rules": [
    { "type": "financial", "factor": "pe", "min": 0, "max": 30 },
    { "type": "sector", "value": ["银行", "证券"] },
    { "type": "technical", "factor": "rsi", "period": 14, "threshold": 30 }
  ],
  "logic": "AND",                          // AND / OR
  "stocks": ["600000", "600036", ...],    // 可选，全市场筛选时为空
  "date": "2026-05-06"                    // 可选，默认当日
}
响应：{
  "code": 0,
  "data": {
    "matched": ["600036", "600000"],
    "total_scanned": 5000,
    "match_rate": 0.04,
    "filters_applied": 3
  }
}
```

### 2. 因子支持范围
按以下因子分类实现筛选逻辑：

| 分类 | 因子 | 参数 | 数据来源 |
|------|------|------|---------|
| 财务指标 | pe（市盈率） | min, max | data_indicator / akshare |
| 财务指标 | pb（市净率） | min, max | data_indicator / akshare |
| 财务指标 | market_cap（市值） | min, max | data_indicator / akshare |
| 板块 | sector（行业板块） | value: [] | data_sector |
| 板块 | concept（概念题材） | value: [] | data_sector |
| 技术指标 | ma（均线） | period, cross_type | 计算生成 |
| 技术指标 | macd | fast/slow/signal | 计算生成 |
| 技术指标 | rsi（相对强弱） | period, threshold | 计算生成 |

### 3. 策略条件解析服务
解析前端策略编辑器输出的 JSON 结构，转换为可执行的筛选参数：

```
POST /data/v1/filter/parse
Body: {
  "stock_filter": {
    "pe": { "min": 0, "max": 30 },
    "sector": ["银行"]
  },
  "timing": { "type": "ma_cross", "period": 20 },
  "risk": { "max_position": 0.3, "stop_loss": 0.05 }
}
响应：{
  "code": 0,
  "data": {
    "parsed_rules": [...],    // 标准化 rules 数组
    "validation": "ok",       // 或错误信息
    "estimated_stocks": 120   // 预计筛选结果数量
  }
}
```

### 4. 与 Go 后端联调
- Go 后端 TC-M3-02 中 POST /api/v1/strategy/:id/submit 触发本接口调用
- **调用路径**：`Go后端 → POST http://localhost:8081/data/v1/filter/`
- **超时时间**：5 秒（TC-M3-02 要求），超时返回 code=50001
- **返回格式**：与 TC-M3-02 中定义的响应格式一致

## 审核点
1. POST /data/v1/filter/ + 财务指标规则 → 返回 matched 列表
2. POST /data/v1/filter/ + 板块规则 → 返回该板块股票列表
3. POST /data/v1/filter/ + 技术指标规则 → 返回计算后结果
4. 多规则 AND 组合 → 只有全部满足才入选
5. 多规则 OR 组合 → 任一满足即入选
6. 响应时间计时：p95 ≤ 200ms（用 Python time.time() 打印日志）
7. 超时场景：请求超时 5 秒 → 返回 code=50001
8. /data/v1/filter/parse：输入前端 JSON → 输出标准化的 rules + estimated_stocks
