## 六、技术债务 & 建议（对审核官说）

### 6.1 需联调确认

1. **后端接口格式**：目前 `backend.ts` 按 `response.data = { code, message, data }` 统一响应格式处理，与后端确认实际格式是否匹配。
2. **WebSocket 事件类型**：需确认后端发送的事件名（`type` / `event`）与前端 `websocket.ts` 中的 `on(event)` 注册一致，避免消息无法分发。
3. **登录 API**：确认 `/api/v1/auth/login` 返回的 token 字段名（`token` / `access_token`），目前写在 auth store 中。
4. **JWT 过期处理**：`code: 40102 / 40103` 由前端判断后跳转登录，需确认后端返回的实际 code 值。

### 6.2 部署前必做

1. **替换 `.env.production` 中的域名**：将 `your-domain.com` 替换为实际域名/IP。
2. **后端 API 联调**：前端所有 API 调用在本地开发环境通过 Vite proxy 代理到 `localhost:8080`，生产部署后需改为实际后端地址。
3. **真机测试**：任务卡中要求的 iOS Safari / Android Chrome 真机验收、Lighthouse 评分均需用户手动执行，前端代码已完成但未实际验证。
4. **Nginx 启动**：需在服务器上安装 Nginx 并执行 `nginx -c D:\ZeroQuant2\deploy\nginx.conf` 启动。

## 七、踩坑记录（技术经验）

| # | 问题 | 解决方案 |
|---|------|----------|
| 1 | Vite 8 + rolldown：`manualChunks` 对象形式报错 | 改用函数形式 `manualChunks(id)` |
| 2 | Vite 8：`target: 'es2015'` + `minify: 'esbuild'` 导致 esbuild 找不到包 | 去掉 minify，使用 rolldown 默认压缩 |
| 3 | vite-plugin-compression：Vite 8 下 .gz 文件路径错误 | 移除插件，改用 Nginx 原生 gzip |
| 4 | pnpm 项目中不可混用 npm 安装依赖 | 会导致 `Cannot read properties of null` 错误 |
| 5 | PowerShell 中文用户名导致 `-File` 参数失效 | 改用 `cmd /c` 调用 |
| 6 | src/backend 是嵌入 git 仓库，当 submodule 处理 | 不要在 frontend 中操作，避免 git 冲突 |
