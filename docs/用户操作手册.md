## 一、系统简介

**ZeroQuant 2.0** 是一款面向 A 股的 B/S 架构全自动量化交易系统。

**目标用户**：有交易经验但不会编程的投资者

**核心功能**：图形化策略编辑 → 数据筛选 → 模拟/实盘自动交易

---

## 二、功能架构

### 三大端

| 端 | 技术 | 端口 | 功能 |
|----|------|------|------|
| Go 后端 | Gin | 8080 | 业务逻辑、策略引擎、交易 |
| Python 数据服务 | FastAPI | 8081 | 行情、K线、盘口、筛选 |
| Vue 前端 | Element Plus | 5173 | 用户界面 |

### 数据流

```
用户（前端） → 策略编辑 → Go后端 → Python数据服务 → 筛选结果
                                        ↓
                              实时行情/盘口 ← 数据源（akshare）
```

---

## 三、快速开始

### 1. 启动服务

```bash
# 终端1：启动 Go 后端（端口 8080）
cd src/backend
go run cmd/server/main.go

# 终端2：启动 Python 数据服务（端口 8081）
cd src/data
uvicorn app.main:app --port 8081

# 终端3：启动 Vue 前端（端口 5173）
cd src/frontend
npm run dev
```

### 2. 访问系统

浏览器打开：`http://localhost:5173`

## 四、用户操作指南

### 4.1 注册与登录

1. 访问系统首页
2. 点击"注册"创建账号
3. 登录后进入主界面

### 4.2 实时行情（新增）

**路径**：侧边栏 → 行情

功能：
- 输入股票代码查询盘口
- 热门股票快速选择（浦发银行、贵州茅台、平安银行、宁德时代）
- 五档买卖盘展示（买1~买5、卖1~卖5）
- 数量进度条可视化
- 自动刷新（可选）

**接口**：`GET /data/v1/orderbook/{stock_code}`

### 4.3 策略管理

**路径**：侧边栏 → 策略管理

功能：
- 查看已创建策略列表
- 策略状态（运行中/已停止）
- 编辑/删除策略

### 4.4 策略编辑器

**路径**：策略管理 → 新建策略

功能：
- 可视化条件组合
- 多因子筛选（财务指标、板块、技术指标）
- 条件设置：PE、PB、市值、均线、MACD、RSI 等

**筛选因子**：

| 类型 | 因子 | 参数 |
|------|------|------|
| 财务 | PE（市盈率） | min, max |
| 财务 | PB（市净率） | min, max |
| 财务 | 市值 | min, max |
| 板块 | 行业板块 | value[] |
| 板块 | 概念题材 | value[] |
| 技术 | MA（均线） | period, cross_type |
| 技术 | MACD | fast, slow, signal |
| 技术 | RSI | period, threshold |

**接口**：`POST /data/v1/filter`

### 4.5 回测

**路径**：侧边栏 → 回测

功能：
- 选择策略
- 设置回测参数（时间范围、资金）
- 查看回测结果

**接口**：`POST /data/v1/filter` → 策略执行

### 4.6 持仓

**路径**：侧边栏 → 持仓

功能：
- 当前持仓列表
- 持仓市值、盈亏展示
- 持仓占比

### 4.7 订单

**路径**：侧边栏 → 订单

功能：
- 订单列表（全部/成交/待成交）
- 订单状态筛选
- 订单详情

### 4.8 仪表盘

**路径**：侧边栏 → 仪表盘

功能：
- 总资产
- 今日盈亏
- 持仓数量
- 运行策略数

## 七、常见问题

### Q1: 行情数据从哪里来？

A: 当前使用模拟数据。生产环境需配置 akshare（免费）或 Tushare Pro（付费）访问真实行情。

### Q2: 为什么盘口数据是模拟的？

A: akshare 访问外部数据源（东方财富）需要网络畅通。当前环境网络受限，使用 `simulated_level2.py` 生成模拟数据。

### Q3: 如何切换到真实数据？

A: 修改 `src/data/.env`：
```
AKSHARE_ENABLED=true
```

### Q4: 前端端口为什么是 5173？

A: Vite 开发服务器默认端口。生产构建后会部署到标准端口。
