# 妞妞和操作员的工作机制

> 版本：V1.0
> 日期：2026-05-06
> 维护者：零号（PM）委托妞妞记录

---

## 一、工作原则

1. 严格按照任务卡排序执行（TC-T0-01 → TC-M1-01 → ...）
2. 代码部分生成 MD 文件（非直接写代码），由操作员审核后提交
3. 非代码部分直接执行
4. 每次必须等操作员说「开始」才开展新工作

---

## 二、循环流程

```
1. 妞妞完成当前任务 → 问操作员是否可以开展下一步
2. 操作员说「开始」→ 妞妞读取新任务卡
3. 筛选内容分类：代码工作 / 非代码工作
4. 触发 task-queue skill → 更新 TASKS.MD，把工作加入队列
5. 代码部分：生成独立 MD 文件放到「桌面/给用户的/」
6. 非代码部分：直接执行
7. 操作员部分完成 → 放指定路径让妞妞审核 → 通过 → 提交 NAS + 本地备份
8. 妞妞部分完成 → 告诉操作员 → 操作员确认「任务已完成」
9. 更新 TASKS.MD
10. 循环
```

---

## 三、角色分工

| 角色 | 职责 |
|------|------|
| 操作员 | 提供需求、审核代码、写文档、确认完成 |
| 妞妞（AI）| 执行开发任务、维护代码架构、产出技术文档 |

---

## 四、任务卡格式

### 开发任务
- 交付物：代码文件（Go/Python/Vue/配置文件等）
- 审核点：功能完整性、代码规范、是否可用

### 管理类任务
- 交付物：文档、表格、会议记录
- 审核点：内容完整、格式规范

## 六、状态标记

| 状态 | 含义 |
|------|------|
| ⏳ 待开始 | 等待操作员说「开始」 |
| 🔄 进行中 | 正在执行 |
| ✅ 已完成 | 任务完成，等待操作员确认 |
| ⚠️ 阻塞 | 遇到问题，需要操作员协助 |
| 📝 审核中 | 等待操作员审核 |

---

## 七、异常处理

### 遇到问题
- 阻塞卡格式：见 DoneCard-Block.md
- 写完之后继续轮询，等待操作员响应

### 审核不通过
- 操作员写反馈，说明问题
- 修复后重新提交

---

## 八、常用操作

| 操作 | 命令 |
|------|------|
| 查看任务卡 | `Get-ChildItem Z:\ZeroQuant2\tasks\niuniu\inbox\` |
| 查看已完成 | `Get-ChildItem Z:\ZeroQuant2\tasks\niuniu\done\` |
| 提交 NAS | 复制文件到对应目录 |
| 提交本地备份 | 复制到 D:\ZeroQuant2-backup\ZeroQuant2\ |

---

## 九、注意事项

1. 代码部分必须生成 MD 文件，由操作员确认后才能提交
2. 禁止直接修改 NAS 上的代码文件
3. 所有交付物必须先本地备份，再提交 NAS
4. 审核通过后，妞妞负责更新 TASKS.MD
