## 一、项目定位

**ZeroQuant 2.0** — 面向 15 人小圈子的 A 股全自动交易系统，B/S 架构。

**目标：** 让有交易经验但不会编程的用户，通过图形化界面（PC + 移动端）自主编写量化策略，系统自动抓取盘口数据、按用户自定义规则分析筛选、全自动执行交易。

**Phase 1 核心闭环：**
```
盘口数据抓取 → 用户自定义规则筛选 → 多因子综合分析 → 模拟交易执行
```

## 五、开发流程（五阶段闭环）

```
需求生成 → 代码编写 → 初步审核 → 终审确认 → 归档备份
```

### 阶段 1：需求生成（零号）
1. 在 `tasks/{终端}/inbox/` 创建需求文档：`REQ-{编号}-{简述}.md`
2. 文档头部包含时间戳和截止时间
3. QQ 通知用户"新需求已发布"

### 阶段 2：代码编写（用户 via Claude Code）
1. 用户读取需求文档
2. 用 Claude Code 编写代码
3. 代码提交到 NAS：对应 src/ 目录
4. 用户 QQ 通知零号"编完了"

### 阶段 3：初步审核（对应终端）
1. 对照需求文档审核代码
2. 在 `tasks/{终端}/done/` 创建审核报告：`REV-{编号}.md`
3. 有问题 → 零号创建修复任务卡 → 回到阶段 2
4. 通过 → 进入终审

### 阶段 4：终审确认（零号）
1. 架构合规性、安全、代码规范审查
2. 在 `tasks/{终端}/inbox/` 创建终审结果通知

### 阶段 5：归档备份（零号）
1. 代码归档到 `backups/`
2. 本地备份到 `D:\ZeroQuant2\backups\{日期}\`
3. GitHub 推送（Uncle0206061/zeroquant2）
4. 更新项目记忆文档

## 七、业务规则（v2.2 新增）

### 7.1 策略编辑器（表单勾选 + 填写）

**编辑器流程：** 新建 → 选择因子 → 配置条件 → 逻辑组合 → 保存 → 回测 → 上线运行

**支持因子类型：**
- 基础行情：最新价、涨跌幅、成交量、成交额、换手率、量比、涨停价、跌停价
- 盘口因子：买一/卖一量、五档总量、委比、委差、盘口厚度
- 技术因子：MA5/MA10/MA20、MACD 金叉/死叉、KDJ、RSI、BOLL 上下轨
- 资金因子：主力净流入、超大单净流入

**规则组合：** AND / OR / NOT 嵌套，最多 10 层条件

**触发方式：** 实时触发、定时触发、K 线闭合触发

**执行模式：** 仅提醒 / 模拟自动交易 / 实盘自动交易（开关分离）

**版本管理：** 支持策略版本管理，可回滚到历史版本

### 7.2 交易风控规则（强制生效）

| 规则 | 值 |
|------|-----|
| 单只个股最大仓位 | 30% |
| 单日最大总亏损 | 5% |
| 止盈 | 固定（5%/10%/自定义）、动态止盈 |
| 止损 | 固定（3%/5%/自定义）、动态止损 |
| 单日最大交易次数 | 20 次 |
| 禁止交易 | ST 股、*ST 股、退市整理股、未开板新股 |

### 7.3 回测规则

| 规则 | 值 |
|------|-----|
| 支持周期 | 1 日 / 1 周 / 1 月 / 3 月 / 1 年 / 自定义区间 |
| K 线复权 | 默认前复权，支持后复权 / 不复权 |
| 回测粒度 | 1 分钟 K 线、日 K |
| 滑点模拟 | 默认 0.1%，可配置 |
| 手续费 | 买入 0.03%，卖出 0.13%（含印花税） |

## 十、异常处理规则（v2.2 新增）

| 场景 | 处理方式 |
|------|---------|
| 主数据源断开 | 1 秒内自动切换备用源 |
| 全部源失效 | 停止交易，推送告警，不执行任何策略 |
| 策略执行失败 | 不重试，标记失败并推送告警 |
| 服务崩溃 | 容器自动重启，重启后恢复策略状态 |
| 数据库异常 | 熔断保护，停止写入，保护数据 |

## 十四、风险与应对（v2.2 新增）

| 风险 | 应对方式 |
|------|----------|
| Level 2 数据源不稳 | 多源备份，1 秒自动切换 |
| WebSocket 压力 | Redis 发布订阅 + 连接池 |
| 数据库异常 | 熔断保护，停止写入 |

### 业务风险

| 风险 | 应对方式 |
|------|----------|
| 策略执行时效 | 模拟盘充分测试后上线 |
| 用户数据安全 | 敏感信息脱敏 + AES 加密 |
| 异常交易 | 自动冻结，禁止自动恢复 |
