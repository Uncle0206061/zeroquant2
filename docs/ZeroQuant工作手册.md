## 一、项目定位

**ZeroQuant 2.0** — 面向小圈子的 A 股量化辅助决策平台，B/S 架构。

**MVP 功能：**
- 行情展示（实时 / 历史 K 线）
- 自选股管理
- 模拟交易（Paper Trading）
- 简单回测

**不做什么：**
- ❌ 实盘交易（需牌照）
- ❌ 复杂策略回测引擎
- ❌ 实时推送行情（MVP 用轮询）
- ❌ 多市场支持（只做 A 股）

## 四、开发流程（五阶段闭环）

```
需求生成 → 代码编写 → 初步审核 → 终审确认 → 归档备份
```

### 阶段 1：需求生成

**执行者：零号**

1. 零号在 `tasks/{终端}/inbox/` 创建需求文档：`REQ-{编号}-{简述}.md`
2. 文档头部必须包含时间戳和截止时间
3. 零号 QQ 通知用户"新需求已发布"

### 阶段 2：代码编写

**执行者：用户（通过 Claude Code）**

1. 用户读取需求文档
2. 用 Claude Code 编写代码
3. 代码提交到 NAS：`src/backend/` 或 `src/frontend/` 或 `src/data/`
4. 用户 QQ 通知零号"编完了"
5. 零号在 `tasks/{终端}/inbox/` 创建交付通知

### 阶段 3：初步审核

**执行者：妞妞 / 大聪明 / 数据官**

1. 对照需求文档审核代码
2. 在 `tasks/{终端}/done/` 创建审核报告：`REV-{编号}.md`
3. 如果有问题：描述问题 → 零号创建修复任务卡 → 回到阶段 2
4. 如果通过：进入终审

### 阶段 4：终审确认

**执行者：零号**

1. 架构合规性审查
2. 安全审查
3. 代码规范审查
4. 在 `tasks/{终端}/inbox/` 创建终审结果通知

### 阶段 5：归档备份

**执行者：零号**

1. 代码归档到 `backups/` 目录
2. 本地备份到 `D:\ZeroQuant2\backups\`
3. GitHub 云备份（推送到 Uncle0206061/zeroquant2 私有仓库）
4. 更新项目记忆文档

## 五、任务卡格式

### 任务卡（零号 → 终端）

```markdown
---
id: task-{终端}-{编号}
type: requirement | fix | notice
priority: high | medium | low
created: 2026-05-02T20:00:00+08:00
deadline: 2026-05-03T12:00:00+08:00
---

## 任务标题

### 背景
（为什么有这个任务）

### 要求
（具体要做什么）

### 验收标准
- [ ] 标准 1
- [ ] 标准 2

### 参考资料
- 文档链接
- 示例代码
```

```markdown
---
task: task-{终端}-{编号}
type: done | issue | question
created: 2026-05-02T20:30:00+08:00
---

## 内容
（描述完成情况 / 问题详情 / 提问）
```
